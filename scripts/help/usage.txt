BioYoda v0.2.0 - Biomedical data processing pipeline

Usage: bioyoda.sh <command> [options]

COMMANDS:
    start                     Start Qdrant server
    stop                      Stop Qdrant server (or a running pipeline)
    build <collection> <stg>  Build/refresh a Qdrant collection (chunk/embed/ingest/insert/...)
    compounds <subcommand>    Compound tooling (FPSim2 / FTO / prediction)
    qdrant <subcommand>       Manage Qdrant vector database
    status                    Show pipeline status
    validate [module]         Validate module outputs (default: qdrant)
    clean [module]            Clean intermediate files
    test [--pipeline]         Run test suite
    push <module>             Sync data/code to Google Drive (for Colab GPU)
    pull <module>             Pull processed results from Google Drive
    help                      Show this help message
    version                   Show version information

ORCHESTRATION:
    Snakemake and the *-update Enju DAGs are retired. Collections are built with plain
    bash via `bioyoda.sh build <collection> <stage> [--delta]` (collections: compounds |
    text | reference | proteins | trials). GPU stages run on a pod (POD_HOST/POD_PORT/POD_KEY).
    The one remaining Enju workflow is diamond-update (standalone DIAMOND protein-similarity
    data-prep with no build stage), run via the `enju` binary directly.

SERVED COLLECTIONS:
    patent_compounds          Patent compounds (alias -> patent_compounds_v2)
    chembl                    ChEMBL reference ligands
    clinical_trials_medcpt    Clinical trials (MedCPT)
    patents_text              Patent text (alias -> patents_text_medcpt)
    esm2                      Protein embeddings (ESM-2)

QDRANT SUBCOMMANDS:
    start                     Start Qdrant server (docker compose)
    stop                      Stop Qdrant server
    restart [path]            Restart with existing storage (auto-detect or specify path)
    status                    Check server status and collections
    reindex <collection>      Trigger HNSW index build after a bulk build
    rebuild <collection>      Rebuild a collection from its profile

QDRANT START OPTIONS:
    --runtime <hours>         Runtime in hours (default: 48)
    --memory <mb>             Memory in MB (default: 32000)
    --out-dir <path>          Use existing storage directory

PUSH/PULL OPTIONS:
    --code-only, -c           Only sync scripts (fast, for code changes)
    --dry-run, -n             Show what would be synced without syncing
    --bg, -b                  Run in background

EXAMPLES:
    # Quick start/stop servers
    bioyoda.sh start
    bioyoda.sh stop

    # Build / refresh collections
    bioyoda.sh build compounds all          # patent_compounds (chunk/predict/ingest/...)
    bioyoda.sh build trials insert          # clinical_trials_medcpt
    bioyoda.sh build reference chembl       # chembl
    bioyoda.sh build patents-text insert    # patents_text (-> patents_text_medcpt)
    bioyoda.sh build proteins insert        # esm2

    # Incremental refresh
    bioyoda.sh build compounds all --delta   # Only process new/changed inputs

    # Qdrant operations
    bioyoda.sh qdrant start                                  # Start locally
    bioyoda.sh qdrant start --out-dir /scratch/qdrant_data  # Use existing storage
    bioyoda.sh qdrant restart                                # Auto-detect and restart
    bioyoda.sh qdrant reindex patent_compounds --monitor     # Trigger + monitor HNSW build
    bioyoda.sh qdrant status
    bioyoda.sh qdrant stop

    # Testing
    bioyoda.sh test                    # Fast fixture mode
    bioyoda.sh test --pipeline         # Full E2E test

    # Google Drive sync (for Colab GPU processing)
    bioyoda.sh push patents                     # Push data + code to Drive
    bioyoda.sh push patents --code-only         # Push only scripts (fast)
    bioyoda.sh push pubmed --dry-run            # Preview what would sync
    bioyoda.sh pull patents                     # Pull processed results

    # Maintenance
    bioyoda.sh status
    bioyoda.sh validate qdrant
    bioyoda.sh stop pubmed --clean
